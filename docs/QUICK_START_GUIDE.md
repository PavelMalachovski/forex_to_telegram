# Быстрый старт - Управление данными Forex Bot

## Краткое описание

Созданы два основных скрипта для управления данными:

1. **`clean_database.py`** - Очистка базы данных (сохраняет пользователей)
2. **`bulk_load_data.py`** - Массовая загрузка данных с ForexFactory

## Быстрые команды

### Загрузка данных за июнь 2024 (как в задаче)
```bash
python bulk_load_data.py --from 2024-06-01 --to 2024-06-30
```

### Загрузка данных с 1 января 2025 до сегодня
```bash
python bulk_load_data.py --from 2025-01-01 --to today
```

### Полная переинициализация базы данных
```bash
# 1. Очистить все данные (кроме пользователей)
python clean_database.py --confirm

# 2. Загрузить данные с начала года
python bulk_load_data.py --from 2025-01-01 --to today
```

### Безопасная проверка перед очисткой
```bash
# Посмотреть что будет удалено
python clean_database.py --dry-run

# Если все в порядке, выполнить очистку
python clean_database.py --confirm
```

## Основные возможности

### clean_database.py
- ✅ Удаляет все новости и логи скрапинга
- ✅ Сохраняет всех пользователей и их настройки
- ✅ Пересоздает справочные данные (валюты, уровни воздействия)
- ✅ Требует явного подтверждения
- ✅ Режим предварительного просмотра (`--dry-run`)

### bulk_load_data.py
- ✅ Загружает данные за любой период
- ✅ Показывает прогресс-бар
- ✅ Перезаписывает существующие данные (по умолчанию)
- ✅ Режим без перезаписи (`--no-overwrite`)
- ✅ Повторные попытки при ошибках
- ✅ Детальная статистика и логирование

## Примеры использования

### Сценарий 1: Первоначальная загрузка данных за июнь
```bash
# Загрузить данные за июнь 2024
python bulk_load_data.py --from 2024-06-01 --to 2024-06-30

# Результат: ~600-900 событий за месяц
# Время выполнения: ~4-6 минут
```

### Сценарий 2: Обновление данных за последнюю неделю
```bash
# Загрузить данные за последние 7 дней
python bulk_load_data.py --from $(date -d '7 days ago' +%Y-%m-%d) --to today
```

### Сценарий 3: Загрузка данных с начала 2025 года
```bash
# Очистить базу данных
python clean_database.py --confirm

# Загрузить данные с 1 января 2025
python bulk_load_data.py --from 2025-01-01 --to today
```

### Сценарий 4: Безопасное добавление новых данных
```bash
# Загрузить только новые данные (не перезаписывать существующие)
python bulk_load_data.py --from 2024-06-01 --to 2024-06-30 --no-overwrite
```

## Мониторинг и логирование

### Лог-файлы
Каждый запуск создает лог-файл с временной меткой:
- `clean_database_YYYYMMDD_HHMMSS.log`
- `bulk_load_YYYYMMDD_HHMMSS.log`

### Просмотр последних логов
```bash
# Последний лог очистки
ls -t clean_database_*.log | head -1 | xargs cat

# Последний лог загрузки
ls -t bulk_load_*.log | head -1 | xargs cat
```

### Статистика в базе данных
Все операции загрузки записываются в таблицу `scraping_logs`:
```sql
SELECT * FROM scraping_logs ORDER BY created_at DESC LIMIT 5;
```

## Производительность

### Типичные показатели
- **Время на один день**: 5-8 секунд
- **События в день**: 15-30 (в среднем)
- **Месяц данных**: 3-6 минут
- **Размер данных**: ~1-2MB на месяц

### Оптимизация
- Используйте меньшие диапазоны дат для быстрого тестирования
- Запускайте в фоновом режиме для больших периодов:
  ```bash
  nohup python bulk_load_data.py --from 2024-01-01 --to today > load.log 2>&1 &
  ```

## Устранение неполадок

### Частые проблемы

1. **Ошибка "No module named 'app'"**
   ```bash
   # Убедитесь что запускаете из корневой директории
   cd ~/forex_bot_postgresql/
   ```

2. **Медленная загрузка**
   ```bash
   # Проверьте интернет-соединение и уменьшите период
   python bulk_load_data.py --from 2024-06-01 --to 2024-06-07
   ```

3. **Ошибки скрапинга**
   ```bash
   # Используйте подробное логирование
   python bulk_load_data.py --from 2024-06-01 --to 2024-06-07 --verbose
   ```

## Безопасность

### Резервное копирование
```bash
# Создать резервную копию перед очисткой
cp forex_bot.db forex_bot.db.backup.$(date +%Y%m%d)
```

### Восстановление
```bash
# Восстановить из резервной копии
cp forex_bot.db.backup.YYYYMMDD forex_bot.db
```

## Автоматизация

### Ежедневное обновление (cron)
```bash
# Добавить в crontab для ежедневного обновления в 4:00
0 4 * * * cd /path/to/forex_bot_postgresql && python bulk_load_data.py --from $(date -d '1 day ago' +\%Y-\%m-\%d) --to today
```

### Еженедельная очистка и полная загрузка
```bash
# Каждое воскресенье в 2:00
0 2 * * 0 cd /path/to/forex_bot_postgresql && echo "YES" | python clean_database.py --confirm && python bulk_load_data.py --from 2025-01-01 --to today
```
