# Отчет об исправлении проблемы с путями к папке logs

## Проблема
Приложение успешно собиралось в Docker, но падало при запуске с ошибкой:
```
"Application failed: [Errno 2] No such file or directory: '/home/ubuntu/forex_bot_postgresql/logs'"
```

Проблема заключалась в использовании абсолютных путей к папке logs, которые не существовали в Docker контейнере.

## Выполненные исправления

### 1. Исправлены основные Python файлы
- **main.py** - заменены абсолютные пути на относительные с поддержкой переменной окружения LOG_DIR
- **enhanced_main.py** - аналогичные исправления
- **production_scheduler.py** - аналогичные исправления

### 2. Исправлены скрипты в папке scripts/
- **health_check.sh** - использует относительные пути от расположения скрипта
- **watchdog.sh** - использует относительные пути от расположения скрипта  
- **monitor.sh** - использует относительные пути от расположения скрипта
- **start_with_monitoring.sh** - использует относительные пути от расположения скрипта
- **render_resource_check.sh** - исправлены абсолютные пути
- **install_service.sh** - исправлены абсолютные пути

### 3. Исправлены тестовые файлы
- **tests/test_comprehensive_fix.py** - заменен абсолютный путь на относительный
- **tests/test_tomorrow_fix.py** - заменен абсолютный путь на относительный

### 4. Обновлен Dockerfile
- **docker/Dockerfile.production** - добавлена переменная окружения LOG_DIR=/app/logs
- Добавлено создание папки logs с правильными правами доступа

## Ключевые изменения в коде

### Логика определения путей
```python
# Старый код (абсолютный путь)
log_dir = Path('/home/ubuntu/forex_bot_postgresql/logs')

# Новый код (относительный путь с поддержкой переменной окружения)
project_root = Path(__file__).resolve().parent
log_dir = Path(os.getenv('LOG_DIR', project_root / 'logs'))
```

### Логика в скриптах bash
```bash
# Старый код
LOG_FILE="/home/ubuntu/forex_bot_postgresql/logs/health_check.log"

# Новый код
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_DIR="$(dirname "$SCRIPT_DIR")"
LOG_DIR="${LOG_DIR:-$PROJECT_DIR/logs}"
LOG_FILE="$LOG_DIR/health_check.log"
```

1. **Портабельность** - код работает как локально, так и в Docker контейнере
2. **Гибкость** - можно переопределить расположение логов через переменную окружения
3. **Безопасность** - нет жестко заданных абсолютных путей
4. **Совместимость** - работает в любой среде развертывания

## Переменные окружения

- **LOG_DIR** - путь к папке для логов (по умолчанию: `{project_root}/logs`)
- **PORT** - порт для Flask API (по умолчанию: 8000)
