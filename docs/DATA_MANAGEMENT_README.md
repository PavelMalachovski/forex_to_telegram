
# Управление данными Forex Bot

Этот документ описывает использование скриптов для управления данными в проекте Forex Bot.

## Обзор скриптов

### 1. clean_database.py
Скрипт для очистки базы данных от всех данных кроме пользователей.

**Что удаляется:**
- Все новости (news_events)
- Логи скрапинга (scraping_logs)
- Справочные данные (currencies, impact_levels) - пересоздаются автоматически

**Что сохраняется:**
- Пользователи (bot_users)
- Настройки пользователей (user_currency_preferences, user_notification_settings)

### 2. bulk_load_data.py
Скрипт для массовой загрузки данных с ForexFactory за указанный период.

**Возможности:**
- Загрузка данных за любой период
- Перезапись существующих данных
- Прогресс-бар и детальная статистика
- Обработка ошибок с повторными попытками
- Подробное логирование

## Установка зависимостей

Перед использованием скриптов убедитесь, что установлены все необходимые зависимости:

```bash
pip install tqdm tenacity python-dotenv
```

## Использование clean_database.py

### Предварительный просмотр
Посмотреть что будет удалено без фактического удаления:

```bash
python clean_database.py --dry-run
```

### Выполнение очистки
Выполнить полную очистку с пересозданием справочных данных:

```bash
python clean_database.py --confirm
```

### Очистка без пересоздания справочников
Очистить данные но не пересоздавать currencies и impact_levels:

```bash
python clean_database.py --confirm --no-reinit
```

### Параметры

- `--dry-run` - Показать что будет удалено без фактического удаления
- `--confirm` - Подтвердить выполнение очистки (обязательный для реального удаления)
- `--no-reinit` - Не пересоздавать справочные данные после очистки

### Безопасность

- Скрипт требует явного подтверждения для выполнения очистки
- При использовании `--confirm` запрашивается дополнительное подтверждение
- Все операции логируются в файл и консоль
- Пользователи и их настройки никогда не удаляются

## Использование bulk_load_data.py

### Загрузка данных за период
Загрузить данные за июнь 2024:

```bash
python bulk_load_data.py --from 2024-06-01 --to 2024-06-30
```

### Загрузка с начала года до сегодня
```bash
python bulk_load_data.py --from 2024-01-01 --to today
```

### Загрузка без перезаписи существующих данных
```bash
python bulk_load_data.py --from 2024-06-01 --to 2024-06-30 --no-overwrite
```

### Подробное логирование
```bash
python bulk_load_data.py --from 2024-06-01 --to 2024-06-30 --verbose
```

### Параметры

- `--from YYYY-MM-DD` - Дата начала (обязательный)
- `--to YYYY-MM-DD|today` - Дата окончания или "today" (обязательный)
- `--no-overwrite` - Не перезаписывать существующие данные
- `--verbose` - Подробное логирование

### Особенности работы

- **Автоматическая перезапись**: По умолчанию существующие данные за те же даты перезаписываются
- **Повторные попытки**: При ошибках скрапинга выполняется до 3 попыток с экспоненциальной задержкой
- **Прогресс-бар**: Показывает текущий прогресс обработки
- **Статистика**: Детальная статистика по завершению
- **Логирование**: Все операции записываются в лог-файл

## Типичные сценарии использования

### 1. Полная переинициализация базы данных
```bash
# Очистить все данные
python clean_database.py --confirm

# Загрузить данные с начала года
python bulk_load_data.py --from 2024-01-01 --to today
```

### 2. Загрузка данных за июнь (как в задаче)
```bash
# Загрузить данные за июнь 2024
python bulk_load_data.py --from 2024-06-01 --to 2024-06-30
```

### 3. Обновление данных за последний месяц
```bash
# Загрузить данные за последние 30 дней
python bulk_load_data.py --from $(date -d '30 days ago' +%Y-%m-%d) --to today
```

### 4. Безопасная проверка перед очисткой
```bash
# Сначала посмотреть что будет удалено
python clean_database.py --dry-run

# Если все в порядке, выполнить очистку
python clean_database.py --confirm
```

## Логирование

Оба скрипта создают лог-файлы с временными метками:
- `clean_database_YYYYMMDD_HHMMSS.log`
- `bulk_load_YYYYMMDD_HHMMSS.log`

Логи содержат:
- Детальную информацию о выполненных операциях
- Статистику обработки
- Информацию об ошибках
- Временные метки всех операций

## Обработка ошибок

### clean_database.py
- Проверяет подключение к базе данных
- Валидирует структуру таблиц
- Использует транзакции для безопасности
- Откатывает изменения при ошибках

### bulk_load_data.py
- Повторные попытки при ошибках скрапинга
- Пропуск проблемных дат с продолжением обработки
- Валидация данных перед вставкой
- Создание записей в scraping_logs для отслеживания

## Мониторинг производительности

### Время выполнения
- **clean_database.py**: Обычно < 1 минуты
- **bulk_load_data.py**: ~2-5 секунд на дату (зависит от количества событий)

### Примерные объемы данных
- Среднее количество событий в день: 20-50
- Размер одной записи: ~1KB
- Месяц данных: ~1-2MB

## Требования к системе

### Минимальные требования
- Python 3.7+
- 100MB свободного места на диске
- Стабильное интернет-соединение для скрапинга

### Рекомендуемые требования
- Python 3.9+
- 1GB свободного места на диске
- Быстрое интернет-соединение

## Устранение неполадок

### Ошибка "No module named 'app'"
```bash
# Убедитесь что запускаете из корневой директории проекта
cd ~/forex_bot_postgresql/
python clean_database.py --dry-run
```

### Ошибка подключения к базе данных
```bash
# Проверьте настройки в .env файле
cat .env | grep DATABASE_URL

# Убедитесь что база данных инициализирована
python init_data.py
```

### Ошибки скрапинга
- Проверьте интернет-соединение
- Убедитесь что ForexFactory доступен
- Попробуйте уменьшить диапазон дат
- Используйте `--verbose` для детального логирования

### Медленная работа
- Уменьшите диапазон дат для bulk_load_data.py
- Проверьте доступное место на диске
- Убедитесь что нет других ресурсоемких процессов

## Безопасность

### Резервное копирование
Перед выполнением clean_database.py рекомендуется создать резервную копию:

```bash
# Для SQLite
cp forex_bot.db forex_bot.db.backup

# Для PostgreSQL
pg_dump your_database > backup.sql
```

### Права доступа
- Скрипты требуют полных прав на базу данных
- Убедитесь что .env файл защищен от чтения другими пользователями
- Не запускайте скрипты от имени root без необходимости

## Поддержка

При возникновении проблем:
1. Проверьте лог-файлы
2. Убедитесь в правильности параметров
3. Проверьте подключение к базе данных
4. Используйте `--dry-run` для безопасного тестирования
5. Используйте `--verbose` для детального логирования
